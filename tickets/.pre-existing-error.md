## Failure

```
1) Grammar-Based SQL Fuzzing
   Error: Timeout of 120000ms exceeded. For async tests and hooks, ensure "done()"
   is called; if returning a Promise, ensure it resolves.
   (C:\projects\quereus\packages\quereus\test\fuzz.spec.ts)
```

All other tests passed (full run took ~8 minutes wall-clock on this agent box).

## Why this is pre-existing / not mine

- **It is a wall-clock timeout, not an assertion failure.** `fuzz.spec.ts` sets a
  per-test mocha budget of `this.timeout(120_000)` and drives `fc.assert(...,
  { numRuns: 100–200 })` with **no fixed seed** — random SQL each run. On a
  loaded box (the whole suite ran ~8 min here) a 200-run generation pass blows
  the 120s budget. The original implementer reported `0 failing` on a faster run.
- **My change cannot reach it.** This ticket only adds an `AbortSignal` `{ signal }`
  option to the query/statement API plus one `throwIfAborted(ctx.signal)` checkpoint
  in the DML drain loop. The fuzzer never passes a `{ signal }`, so `ctx.signal`
  is `undefined` and the new checkpoint is an inert no-op branch. `fuzz.spec.ts`
  is outside my diff entirely.

## Work-around

None needed — no code disabled. The fuzz test was left untouched; it simply needs
more wall-clock than the agent box afforded in one full-suite pass.
