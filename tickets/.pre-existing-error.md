# Pre-existing failure: docs/sql.md over its size ratchet

## Command
From repo root:

```
yarn docs:check      # → node scripts/check-docs.mjs
```

(also runs first inside `yarn check`)

## Failure

```
docs/sql.md: 28750 words exceeds its ratchet of 28657 (+93) — a doc may shrink, never grow
```

Check C (size ratchet) in `scripts/check-docs.mjs`, ratchet recorded in
`docs/.doc-budget.json`.

## Why this is pre-existing (not caused by this ticket)

- `docs/sql.md` is **not** in this ticket's diff — `git status` shows it unmodified,
  and this ticket (`3-runtime-context-shadow-strict`) only touches `docs/runtime.md`
  among docs.
- The committed HEAD version already violates the ratchet: `git show HEAD:docs/sql.md | wc -w`
  reports 28750 words against a recorded ratchet of 28657, so `yarn docs:check` is
  already red on a clean `main` before any of this ticket's changes.

## Fix direction (for the triage pass)

Either shrink `docs/sql.md` back under 28657 words, or — if the growth is intentional
and reviewed — raise its ratchet entry in `docs/.doc-budget.json` (the sanctioned path
is `node scripts/check-docs.mjs --update-ratchet --force`, justified in the commit).
This ticket deliberately did **not** touch `sql.md`'s ratchet entry, since bumping it
would silently absorb a growth this ticket did not author.
