# Pre-existing failure: three docs are over their size ratchet

## Command

```
node scripts/check-docs.mjs      # a.k.a. `yarn docs:check`
```

(run from repo root, `C:\projects\quereus`)

## Failures

```
docs/runtime.md: 13840 words exceeds its ratchet of 13477 (+363) — a doc may shrink, never grow
docs/schema.md: 16029 words exceeds its ratchet of 15690 (+339) — a doc may shrink, never grow
docs/sync.md:   14516 words exceeds its ratchet of 14321 (+195) — a doc may shrink, never grow
```

Check C (the size ratchet) in `scripts/check-docs.mjs`. The three docs each grew past the
word count recorded in `docs/.doc-budget.json` without that entry being lowered/raised.

## Why this is pre-existing (not caused by the docs-vu-split ticket)

- The `docs-vu-split` ticket touched only `docs/view-updateability.md`, five new
  `docs/vu-*.md` files, `docs/.stability.json`, `docs/.doc-budget.json`, and a handful of
  inbound-link labels in `architecture.md` / `lens.md` / `migration.md` /
  `mv-schema-change.md` / `optimizer-fd.md` / `sql.md`.
- `docs/runtime.md`, `docs/schema.md`, and `docs/sync.md` are **unmodified** by this ticket
  — `git diff -- docs/runtime.md docs/schema.md docs/sync.md` is empty, so the working-tree
  content equals `HEAD`. The overage therefore already existed at `HEAD`: `yarn docs:check`
  (and hence `yarn check`) was red before this ticket began.
- The three failing docs are unrelated to view updateability; nothing in this diff changes
  their word counts.

## Suggested resolution (for the triage pass)

Each doc grew without its ratchet being updated. The root cause is a prior change that
edited one of these docs and neither shrank it back under its recorded size nor raised the
ratchet with justification (per `docs/doc-conventions.md § The size ratchet`, a raise needs
`--update-ratchet --force` and a commit-message reason). Either trim the three docs back
under their recorded sizes, or raise the three entries with a justification — do **not**
silently `--force` without recording why.
