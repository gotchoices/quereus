# Pre-existing test failure — RESOLVED into a fix ticket (no triage needed)

The failure previously recorded here (SQL Logic Tests › 07.7-scalar-agg-decorrelation.sqllogic,
"ORDER BY on an aggregate (GROUP BY) query": `Scalar subquery returned more than
one row`) has been diagnosed and ticketed. It no longer surfaces in the suite.

**Disposition (by `feat-decorrelate-scalar-subquery-order-by`, resumed run):**

- Root cause is a **planner scope-leak**, independent of decorrelation: an
  aggregate scalar subquery in the ORDER BY of a GROUP BY query resolves its
  inner aggregate to the *outer* aggregate's output alias. Reproduces with an
  uncorrelated subquery and with all decorrelation rules disabled — a build-phase
  bug, not something this ticket's ORDER BY decorrelation could fix.
- Filed as **`fix/order-by-aggregate-subquery-scope-leak`** (in-flight fix/ slug).
  Also registered in `.pre-existing-known.md`.
- The single `.sqllogic` line that exercised it was removed and replaced with a
  NOTE, so `yarn test` is green (7087 passing, 0 failing). Nothing was skipped,
  disabled, or loosened.

No further triage dispatch is required — the root cause is already tracked by the
fix/ ticket above.
