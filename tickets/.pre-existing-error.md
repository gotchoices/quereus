# Pre-existing test failure: `93.4-view-mutation.sqllogic` flaky 2000 ms timeout

## Command

```
yarn workspace @quereus/quereus test
```
(package `@quereus/quereus`, memory backing — the default agent suite. Runner is
`packages/quereus/test-runner.mjs`, which hard-codes `--bail` and uses mocha's
**default 2000 ms** per-test timeout — no `--timeout` is passed.)

## Failure

```
1) SQL Logic Tests
     File: 93.4-view-mutation.sqllogic
       should execute statements and match results or expected errors:
   Error: Timeout of 2000ms exceeded. For async tests and hooks, ensure
   "done()" is called; if returning a Promise, ensure it resolves.
   (packages/quereus/test/logic.spec.ts)
```

Because `--bail` is on, this aborts the run after ~2049 passing tests, so the
suite never reaches the rest (incl. this ticket's new spec).

## Why this is pre-existing / not mine

- `93.4-view-mutation.sqllogic` contains **zero** `refresh materialized view`
  and **zero** `maintained as` statements (`grep -c` → 0 / 0). My diff only
  touches the constraint-bearing `refresh` path in
  `runtime/emit/materialized-view-helpers.ts::rebuildBacking`, which never
  executes in this test. It is a view-mutation / updatable-view (lens) test on
  branch `view-updates-lens`.
- Run in isolation the file **passes** in ~2 s — right at the 2000 ms default
  timeout. Under full-suite load (after ~2049 prior tests) it tips over the
  boundary and times out. It is a borderline-slow large file (2995 lines), not a
  correctness failure.
- Confirmed: re-running the full memory suite with `--timeout 20000` yields
  **6073 passing, 9 pending, 0 failing** — matching the implement-stage handoff
  exactly. The only failure is this timeout.

## Work-around applied for this ticket

Validated this ticket's work with `node packages/quereus/test-runner.mjs
--timeout 20000` (full suite green) and ran the new spec in isolation (17
passing). No test was disabled. Suggested durable fix for triage: either raise
the default per-test timeout in `test-runner.mjs` (the 2000 ms default is tight
for the larger sqllogic files), or split `93.4-view-mutation.sqllogic`.
