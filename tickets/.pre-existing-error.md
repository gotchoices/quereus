# Pre-existing failure: `@quereus/quoomb-web` lint cannot find an ESLint config

## Command

```
yarn workspace @quereus/quoomb-web lint
# → eslint src/**/*.{ts,tsx}
```

Run from repo root (`C:\projects\quereus`), package `packages/quoomb-web`.

## Failure

```
Oops! Something went wrong! :(
ESLint: 9.39.2
ESLint couldn't find an eslint.config.(js|mjs|cjs) file.
From ESLint v9.0.0, the default configuration file is now eslint.config.js.
```

Exit code 2. ESLint never evaluates any source file — it aborts during config discovery.

## Why this is pre-existing (not caused by ticket `10-sync-drain-host-wiring`)

- No `eslint.config.*` or `.eslintrc*` exists anywhere in the repo — not in
  `packages/quoomb-web/`, not at the root (`ls` of both turned up nothing).
- The failure is a config-**discovery** abort; it is independent of the contents
  of any `.ts`/`.tsx` file, so no source edit could introduce or fix it.
- I did not modify `packages/quoomb-web/package.json` (the `lint` script is
  unchanged — `git status --porcelain` shows it clean).
- `AGENTS.md` § Build & Test states "Only `packages/quereus` has a lint script",
  so the quoomb-web `lint` script appears vestigial / unmaintained under ESLint v9.

## Work-around taken for this ticket

None needed — the lint script does not gate the code. I validated the ticket's
changes with `yarn workspace @quereus/quoomb-web typecheck` (strict `tsc --noEmit`,
**passes**) and `yarn workspace @quereus/quoomb-web test` (Vitest, **passes**,
including the new `sync-maintenance.test.ts`). The lint command was not disabled or
altered.
