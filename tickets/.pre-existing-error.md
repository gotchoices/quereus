## Why this is pre-existing (not from this ticket)
- It fails **only in store (LevelDB) mode**; the full memory-backed suite is green
  (`node test-runner.mjs` → 4175 passing, 9 pending, 0 failing).
- It is in **materialized-view row-time maintenance** (`dml-executor.ts processUpdateRow`,
  table `ltc`), a subsystem this ticket never touches. This ticket
  (`lens-multi-source-put-insert-fanout`) adds only the decomposition-lens INSERT
  fan-out in `planner/mutation/decomposition.ts`, `planner/building/view-mutation-builder.ts`,
  `planner/nodes/view-mutation-node.ts`, and `runtime/emit/view-mutation.ts`. No
  decomposition lens is exercised by `53-materialized-views-rowtime.sqllogic`.
- **Verified by `git stash`**: with all of this ticket's changes stashed (clean tree at
  the same SHA), `node test-runner.mjs --store` reproduces the *identical* failure
  (`53-materialized-views-rowtime.sqllogic: UNIQUE constraint failed: ltc PK`). So it
  is not introduced by this ticket.

## Work-around taken
None needed — the failure is in a separate subsystem and does not block this ticket's
own validation. This ticket's tests (`lens-put-fanout.spec.ts`, `lens-advertisement.spec.ts`)
pass in both memory and store mode; the rest of the store suite passes (1077 passing,
3 pending, this 1 failing).
