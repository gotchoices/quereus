## Why pre-existing

Verified directly: `git stash -u` (removing every change of this ticket) and
re-running `yarn workspace @quereus/store run test` produces the **identical**
2 failures at HEAD (`9f68af8e`). Root cause is the known, already-ticketed
implicit-`select *` table-form round-trip gap: the canonical
`create table … maintained as` persistence form cannot distinguish an implicit
`select *` body from an explicit rename list, so a source widened between
sessions makes the re-planned body arity-error against the persisted declared
column list instead of refilling. **Already filed and analyzed in
`tickets/backlog/mv-table-form-implicit-columns-roundtrip.md`** (these two tests
are listed there verbatim as the residuals) — no new ticket is needed; promotion
of that backlog item is the fix.
