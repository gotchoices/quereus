## What this is

The split in `docs-vu-split` moved most of `docs/view-updateability.md` into four
satellites but left the original file in place as an overview. About 33 source files carry
`docs/view-updateability.md § <Section>` comments (~60 lines); for every section that
moved, the comment now names the wrong file. This ticket retargets each to the file that
holds the section, per the **section → file map** in `docs-vu-split`.

**This is hygiene, not a build gate.** `scripts/check-docs.mjs` validates the
`docs/foo.md` *path* in a comment, not the `§ <Section>` marker (`bareDocRefs`, and the
comment about trailing prose markers at line ~248). The overview file survives, so every
one of these comments still resolves and the build stays green with or without this pass.
The value is a reader who clicks `docs/view-updateability.md § Inner Join` landing on the
section, not on a stub that bounces them one hop further. Do not manufacture urgency the
checker does not enforce — but do it correctly, because a comment naming the wrong file is
exactly the drift this whole effort exists to kill.

independently.

## How to do it safely

- Enumerate first: `grep -rn "view-updateability.md" packages/*/src` is the worklist.
- Edit **comment text only**. No behavioral code changes — this ticket must not alter a
  single statement, only the doc path inside a comment.
- The heaviest cluster is `§ Authored inverses` (≈20 lines across `mutation/`, `building/`,
  `nodes/`, `analysis/`) → all go to `docs/vu-inverses.md`. The next is `§ Common Table
  Expressions` / `§ Inner Join` / `§ Selection` → `docs/vu-operators.md`.
- Comments already pointing at a section that **stayed in the overview** (`§ The Update Site
  Model`, `§ `returning``, `§ Information Schema Surface`, `§ surface authority`) need **no
  change** — verify, don't edit.

## Edge cases & interactions

- **Do not touch `docs/lens.md § …` halves** in comments that name both docs — only the
  `view-updateability.md` half is in scope; the lens paths are correct.
- **`{@link}` and TSDoc references.** A few refs sit inside `{@link resolveCteTarget}`-style
  or `@param` blocks; retarget the doc path without disturbing the TSDoc tag.
- **Stale section names.** A comment may name a section that the split *renamed or merged*.
  If the `§` text no longer matches any heading in the target file, update the section name
  to the surviving heading too — a comment that names a real file but a dead section is the
  same drift one level down.
- **No new refs to satellites yet.** If `docs-vu-split` already retargeted a few source
  comments opportunistically, they are correct — skip, don't double-edit.
- **Build-only verification is the tell.** Because this is comment-only, `yarn build` must
  pass with zero type changes; if it doesn't, a comment edit escaped its comment.
