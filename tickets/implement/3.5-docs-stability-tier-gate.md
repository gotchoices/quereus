## What this is

The prereq ticket, `docs-stability-tiers`, labels every user-facing doc with a
stability tier and records the assignment in `docs/.stability.json`. That property
holds for exactly one commit unless something enforces it. The next doc to land will
have no banner, and nobody will notice.

`scripts/check-docs.mjs` already runs three checks and is the first, cheapest link in
`yarn check`. This ticket adds a fourth.

The check validates *pointers and shape*, in the same spirit as the existing
comment on Check B ("The checker validates pointers, not semantics"). It cannot tell
you whether lenses really are Experimental. It can tell you that lenses are labelled,
that the label is a tier that exists, and that the label agrees with the map.

## Check D — stability banners

Add to the header comment block, in the same style as A/B/C:

```
   D. Stability tiers — every docs/*.md is classified in `docs/.stability.json`, every
                        tiered doc carries exactly one header banner naming its
                        recorded tier, and every tier named anywhere is one that
                        `docs/stability.md` defines.
```

The banner form, pinned by the prereq ticket:

```markdown
> **Stability: Experimental** — see [Stability Tiers](stability.md#tiers).
```

### What it asserts

Let `docFiles()` be the existing walk (`docs/**/*.md` minus the `EXEMPT` frozen
review artifacts — so `review.md` / `review.html` never reach this check).

1. **Tier vocabulary agreement.** The `###` headings under `## Tiers` in
   `docs/stability.md` are the valid tier names. They must equal the `tiers` array in
   `docs/.stability.json`, as a set. Neither file is allowed to be the sole source —
   they must agree, so renaming a tier in one place fails loudly rather than silently
   half-applying.

2. **Total classification.** Every file in `docFiles()` appears exactly once across
   `.stability.json`'s `docs` keys and `untiered` array. A doc in neither fails
   (`docs/foo.md: not classified in docs/.stability.json — add it to 'docs' with a
   tier, or to 'untiered'`). A doc in both fails. An entry naming a file that no
   longer exists fails — unlike the ratchet, whose orphan entries are deliberately
   inert, a stale tier entry hides a missing classification.

3. **Header banner present and correct.** For each doc in the `docs` map: the first
   banner line must appear within the **header window** — the lines from the `#` H1
   up to the first subsequent heading of any level, or six non-blank lines, whichever
   comes first — and its tier must equal the recorded tier. Missing → fail. Present
   but disagreeing with the map → fail, naming both values. More than one banner in
   the header window → fail.

4. **No banner on an untiered doc.** A doc in `untiered` carrying a banner anywhere
   fails. `docs/stability.md` is itself untiered; a banner on it would be circular.

5. **Section banners are valid, not mapped.** A banner *below* the header window is a
   section override (this is how `declare schema` is Beta inside a Stable `sql.md`).
   It is checked for tier validity against the vocabulary, and nothing else — it is
   deliberately not required to match the doc's recorded tier, since disagreeing with
   the doc's tier is its entire purpose.

6. **The banner's link resolves.** It already does: the banner is an ordinary
   markdown link and Check A resolves `stability.md#tiers` against real headings.
   Do not re-implement that here. Do add a `selfTest()` case pinning `tiers` as the
   slug of `## Tiers`, so a rename of that heading is caught by the slugifier
   self-test rather than by thirty simultaneous dead-anchor failures.

### Implementation notes

- **Reuse the module's existing machinery.** `readText` (CRLF normalization — load
  bearing on win32, do not `readFileSync` directly), `stripFences` (a doc showing the
  banner form inside a fence is an illustration, not a banner — `doc-conventions.md`
  will do exactly this), `repoPath`, `docFiles`, and the shared `fail` collector.
  Wire `checkStability(fail)` into `main()` after `checkInvariants(fail)`.
- **Parse the banner with one anchored regex**, exported as a small pure function so
  `selfTest()` can exercise it without touching the filesystem:
  `/^>\s+\*\*Stability:\s+(\w+)\*\*\s+—\s+see \[Stability Tiers\]\(stability\.md#tiers\)\.$/`
  Note the em dash (`—`, U+2014), matching the invariant register's `code:` /
  `guard:` line style. Return `null` on no match rather than throwing.
- **Do not loosen the regex to tolerate near-misses.** A banner that is one character
  off is a banner that will be copy-pasted wrong forever. Fail with the offending
  line so the author sees the diff.
- **`docs/.stability.json` is read, never written** by this script. There is no
  `--update-stability` counterpart to `--update-ratchet`: classifying a new doc is a
  judgement call, and a flag that auto-classifies is a flag that classifies
  everything `untiered`.
- Update the `docs:check` description in `docs/doc-conventions.md` § The stability
  banner: the convention is now mechanically enforced, and a new doc must be
  classified before `yarn check` passes.

## Edge cases & interactions

- **A new doc is added with no classification.** This is the whole point: it must
  fail, with a message that tells the author the two ways out (`docs` with a tier, or
  `untiered`). Verify by hand with a scratch file, then delete it.
- **A doc showing the banner form inside a fenced code block.** `doc-conventions.md`
  and `stability.md` both will. `stripFences` must run before banner extraction, or
  `stability.md` (untiered) fails rule 4 against its own example. This is the single
  most likely way to get this check wrong.
- **`stability.md` documents the tiers *and* is untiered.** Rules 1 and 4 both touch
  it, from opposite directions. Cover it explicitly in a test.
- **A doc with no H1.** The header window is defined from the H1. A doc lacking one is
  a real defect for a tiered doc (fail: cannot locate the header window); harmless for
  an untiered one (rule 4 still scans the whole file).
- **A banner in the header window of an untiered doc.** Rule 4 fires, not rule 3.
  Make sure the untiered path does not accidentally take the "missing banner" branch.
- **A section banner before the first subsequent heading.** By construction impossible
  — the header window *ends* at the first subsequent heading. Do not add a separate
  line-count-only window; the six-non-blank-line bound is a fallback for a doc whose
  H1 is followed by a long intro before the first `##`.
- **CRLF working tree.** Every regex here assumes `\n`. Read through `readText`. The
  existing module comment says why; heed it.
- **Frozen review artifacts.** `EXEMPT` removes them from `docFiles()` before this
  check sees them. Do not add them to `.stability.json` "for completeness" — rule 2's
  orphan-entry failure will then fire on them.
- **Interaction with Check C.** This check adds no words to any doc, so the ratchet is
  untouched. If you find yourself needing to edit `docs/.doc-budget.json`, you have
  drifted out of scope.

## Expected outputs

- `node scripts/check-docs.mjs` → `Docs OK: links resolve, invariants well-formed, sizes within ratchet.`
  Consider extending that success line to mention tiers; if you do, it is prose, not a
  contract.
- Negative cases, each verified by hand (make the edit, run the checker, confirm the
  message, revert the edit):
  - delete the banner from `docs/lens.md` →
    `docs/lens.md: tiered 'Experimental' but carries no stability banner under its H1`
  - change `docs/lens.md`'s banner to `Stable` →
    `docs/lens.md:2: banner says 'Stable' but docs/.stability.json records 'Experimental'`
  - add a scratch `docs/scratch.md` →
    `docs/scratch.md: not classified in docs/.stability.json`
  - add a banner to `docs/architecture.md` (untiered) →
    `docs/architecture.md:N: untiered doc carries a stability banner`
  - add `"Provisional"` to `.stability.json`'s `tiers` →
    `docs/.stability.json: tier 'Provisional' is not defined in docs/stability.md`
- `selfTest()` gains pure cases for the banner regex: the exact form parses to its
  tier; a wrong dash, a missing link, a lowercase `stability:`, and a banner nested in
  a fence all parse to `null` or are stripped.
- `yarn docs:check` passes; nothing under `packages/` is touched, so `yarn lint` and
  `yarn test` are unaffected. `yarn check` runs `test:full` and is not agent-runnable —
  do not attempt it.
