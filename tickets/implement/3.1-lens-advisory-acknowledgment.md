## Scope

Sits **atop** the advisory list `lens-prover-and-attachment` produces. That ticket
makes the lens write-sound and emits coded warnings into the deploy report; this
ticket lets a developer consciously accept those warnings in source, re-surfaces
them when the underlying facts change, and lets a project escalate specific codes
to hard requirements. No new enforcement semantics — pure governance over the
advisory channel.

## Current state (verified)

- Reserved-tag registry (`packages/quereus/src/schema/reserved-tags.ts`) already
  ships:
  - Template `quereus.lens.ack.<code>` with `valueSchema: 'required-nonempty-rationale'`
    (an empty rationale is validated as `severity:'warning'` — the "empty ack
    rationale is itself a warning" rule, already structurally enforced).
  - Template `quereus.lens.access.<col>`.
  - `validateReservedTags(tags, site)` (policy-free; returns `TagDiagnostic[]`),
    `getReservedTag(tags, key)`, `getReservedTagByTemplate(tags, template): TemplatedTagInstance[]`
    (a `quereus.lens.ack.no-backing-index:vin` tag yields `segment === 'no-backing-index:vin'`).
  - `TagSite` includes `'logical-table'` and `'logical-constraint'`.
- `lens-compiler.ts:validateLensTags(slot)` already runs shape/site validation
  and throws on errors / logs warnings. This ticket reads the **ack semantics**
  through `getReservedTagByTemplate` — `reserved-tags.ts` deliberately reads no
  ack semantics itself ("ack fingerprinting … stays in its owning ticket").
- `lens-prover-and-attachment` produces `LensDiagnostic { code, severity, site, message, fingerprintInputs }`
  and a `LensDeployReport`, and `emitApplySchema` yields advisory rows with a
  stable hook for the acknowledged-count tally. **Build on those exact shapes.**

## Design

`docs/lens.md` § "Acknowledging advisories" is authoritative. Mechanics to lock:

### Coded + sited + targeted
Each advisory has a stable code and a logical site (table + optional constraint/column).
An ack targets one site: `quereus.lens.ack.<code>` on the logical table or
constraint suppresses *that* advisory from the default report. The optional
`:<target>` segment (`quereus.lens.ack.no-backing-index:vin`) narrows to a specific
column/constraint when a table has several instances of one code. Match an
advisory to an ack by `(code, site)`.

### Tally + expand
Deploy summary always reports `acknowledged: N` and expands on demand. The
`LensDeployReport` keeps the full acknowledged list available for tooling. Wire
into the advisory-row surface `emitApplySchema` already emits: acknowledged
advisories are suppressed from the default rows but counted; an expand mode (engine
option or `apply schema X with ...`, or simply a second TVF — decide in implement,
lean to a report field consumed by an introspection TVF) lists them.

### Fingerprinted (the anti-fatigue guarantee)
Compute a fingerprint per acknowledged advisory from `LensDiagnostic.fingerprintInputs`
(constraint columns, presence/absence of a covering structure, a **coarse
cardinality band**, …). When the ack is written, the fingerprint is recorded.
If it changes later (cardinality crosses a band, a covering MV is dropped,
constraint columns evolve), the advisory **re-surfaces as un-acknowledged**,
flagged *"previously acknowledged; situation changed."*

- Fingerprint = stable hash of a canonicalized `FingerprintInputs`. Cardinality
  must be **banded** (e.g. powers of ten / exp buckets) so ordinary row-count
  drift does not churn the fingerprint — only a band crossing does. Document the
  bands.
- **Storage (resolved-lean: in-DDL).** Store the recorded fingerprint alongside
  the ack tag so it survives schema export round-trip, is version-controlled, and
  shows up in review (the same rationale the lens doc keeps acks in-source). Shape
  option: extend the tag value, or a sibling `quereus.lens.ack.<code>.fp` key, or
  encode `rationale|fingerprint` — pick the least-noisy that round-trips through
  `reserved-tags.ts` validation; if a new key shape is needed, add its spec to the
  registry. The out-of-band `.tess/lens-ack-fingerprints.json` sibling is an
  acceptable documented opt-in fallback (risk: silent reset) — do NOT make it the
  default.
- A first-write ack with no recorded fingerprint is honored (record-on-first-sight);
  it only re-surfaces once a recorded fingerprint exists and mismatches.

### Escalation policy
A per-schema tag or engine option promotes specific codes:
- `error-on: [code]` — always a hard error; an ack cannot suppress it.
- `require-ack: [code]` — an un-acknowledged instance errors; a valid acknowledged
  instance clears. The sweet spot for `lens.no-backing-index`: forces a conscious
  decision without blocking a developer who has genuinely accepted the commit-time scan.

**Default policy is empty** (resolved open question): out of the box `error-on: []`,
`require-ack: []` — no code escalated. Projects opt in. Parse the policy from a
per-schema tag (e.g. on the logical schema declaration) or an engine option;
apply it in `lens-compiler.ts` after the prover runs and acks are matched: an
`error-on` code or an un-acked `require-ack` code converts the advisory to a
blocking error (thrown atomically, same path as prover errors).

## Key Tests (TDD)

- **Ack suppresses advisory.** `with tags ("quereus.lens.ack.no-backing-index" = 'low-write table, commit-time scan accepted')` → advisory acknowledged; deploy summary tallies `acknowledged: 1`; default rows omit it; expand lists it.
- **Empty ack rationale is a warning.** `... = ''` → meta-warning (already structural in `reserved-tags.ts`; assert it surfaces through the deploy report, not just logs).
- **Targeted ack.** Two `no-backing-index` instances on one table; `quereus.lens.ack.no-backing-index:email` acks only the email one; the other still surfaces.
- **Fingerprint re-surface.** Acknowledge; then drop the covering MV / change constraint columns / cross a cardinality band → advisory re-surfaces flagged "previously acknowledged; situation changed".
- **Fingerprint stability.** Ordinary row-count drift within a band does NOT re-surface the ack.
- **Escalation `require-ack`.** Set `require-ack: [lens.no-backing-index]`; un-acked instance → deploy errors; valid ack clears.
- **Escalation `error-on`.** Set `error-on: [lens.no-backing-index]`; even a valid ack does NOT suppress — deploy errors.
- **Round-trip.** Ack tag + recorded fingerprint survive schema export → re-import → re-apply without re-surfacing (proves in-DDL storage round-trips).
