## What shipped

A new governance layer atop the prover's advisory channel. The prover (already
shipped) emits coded+sited+fingerprint-bearing `LensDiagnostic` warnings; this
ticket lets a developer accept them in source, re-surfaces them on material
change, and lets a project escalate specific codes to hard errors. **No new
enforcement semantics** — pure governance over the advisory list.

### Mechanics implemented (`src/schema/lens-ack.ts`, new module)

- **Ack matcher.** `quereus.lens.ack.<code>[:<target>]` tags are read off the
  logical table's tags and each constraint's tags via `getReservedTagByTemplate`.
  An advisory matches an ack when the ack's `<code>` equals the advisory code with
  its `lens.` prefix stripped, and (if present) the `:<target>` names the
  advisory's constraint / column / one of its `fingerprintInputs.constraintColumns`.
  An untargeted table ack suppresses the class for that table; an untargeted
  constraint ack suppresses only its own constraint (matched by column set).
- **Fingerprint.** `computeAdvisoryFingerprint(code, site, inputs)` → a 12-char
  base64url FNV-1a digest of the canonicalized `FingerprintInputs` (columns
  lowercased+sorted, cardinality already **banded** by the prover). Bands:
  `empty`/`small`<1e3/`medium`<1e6/`large`/`unknown`.
- **In-DDL fingerprint storage (chosen shape).** The recorded fingerprint rides
  the ack tag *value* as a trailing `#fp=<digest>` token — the least-noisy shape
  that round-trips through `reserved-tags.ts` validation unchanged (value is still
  a single non-empty rationale) and avoids a template collision a sibling
  `.<code>.fp` key would cause. Record-on-first-sight: a bare rationale (no `#fp=`)
  is honored *unconditionally* (never re-surfaces). A recorded fingerprint that
  mismatches the recomputed one re-surfaces the advisory flagged "previously
  acknowledged; situation changed" (`LensDiagnostic.resurfaced = true`, new field).
- **Escalation policy.** `resolveEscalationPolicy(logicalTable)` reads two new
  reserved tags `quereus.lens.policy.{error-on,require-ack}` (CSV of codes,
  default-empty). `applyAckGovernance` converts `error-on` codes to blocking
  errors (ack cannot suppress) and un-acknowledged `require-ack` codes to errors.
  Escalation errors aggregate with the prover's and throw atomically before any
  catalog mutation.
- **Deploy summary + expand.** `LensDeployReport` gained an `acknowledged:
  AcknowledgedAdvisory[]` field (tally = `.length`). The new integrated TVF
  `quereus_lens_advisories(logical_schema)` (`func/builtins/explain.ts`, registered
  in `index.ts`) expands one row per advisory of the last deploy with `status`
  (`active`/`re-surfaced`/`acknowledged`/`acknowledged-unconditional`), rationale,
  and current/recorded fingerprints.

### Wiring

`deployLogicalSchema` (`lens-compiler.ts`) now calls `applyAckGovernance(slot,
prove.warnings, resolveEscalationPolicy(logicalTable))` right after `proveLens`,
folding governance errors into the atomic `proveErrors` throw, governance
warnings into the report, and `acknowledged` onto the report.

## Use cases to exercise during review

- `with tags ("quereus.lens.ack.no-backing-index" = '<rationale>')` on a logical
  table with `unique(col)` over a basis lacking a covering MV → `acknowledged: 1`,
  default report omits `lens.no-backing-index`, TVF lists it.
- Append `#fp=<wrong>` → advisory re-surfaces flagged; append `#fp=<correct>`
  (read from `report.acknowledged[0].currentFingerprint`) → stays acknowledged.
- `quereus.lens.policy.require-ack = 'lens.no-backing-index'` without an ack →
  deploy throws `/require-ack/`; add the ack → clears.
- `quereus.lens.policy.error-on = 'lens.no-backing-index'` with a valid ack →
  deploy still throws `/error-on/`.

## Known gaps / deviations (treat work as a starting point)

- **Escalation policy is per-(logical-table), not schema-wide.** The source ticket
  framed it as "per-schema tag or engine option". There is **no schema-level tags
  AST** (`DeclareSchemaStmt` carries none) and adding an engine option meant
  plumbing into `database.ts`; I chose the in-source per-table reserved tag (most
  consistent with the ack philosophy: version-controlled, review-visible). A
  schema-wide policy is expressed by tagging each table. A global engine-option
  source remains a reasonable future addition — flagged in docs and the shipped
  bullet. **Reviewer: confirm this deviation is acceptable, or file a follow-up for
  a global source.**
- **Cardinality-band re-surface is unit-tested, not integration-tested.** Memory
  tables report a static `estimatedRows`, so a band crossing can't be staged by
  inserting rows in a test. The band logic + fingerprint stability/sensitivity is
  covered by a direct `computeAdvisoryFingerprint` unit test; the integration
  re-surface test uses a **constraint-column change** instead. Likewise the
  "covering-MV dropped → re-surface" path isn't directly integration-tested (with
  a covering MV present the advisory doesn't fire, so there's nothing to ack until
  it's gone). The fingerprint *does* include `hasCoveringStructure` and
  `cardinalityBand`, so the mechanism is in place — only the live-data trigger is
  unexercised.
- **`#fp=` value-suffix parsing** uses an anchored regex `/\s*#fp=([A-Za-z0-9_-]+)\s*$/`.
  A rationale that legitimately *ends* in a `#fp=<base64url-ish>` token would be
  mis-parsed (very low risk; the token is engine-authored). Reviewer may prefer a
  less collision-prone delimiter.
- **Empty-rationale ack still suppresses and still clears `require-ack`** (while
  emitting the meta-warning). Defensible but arguable — one could treat an empty
  rationale as *not* a valid ack for require-ack purposes. Untested edge.
- **Constraint-scoped acks** (ack tag placed on a constraint's `tags` rather than
  the table's) are implemented but only **table-level** acks (incl. `:target`) are
  tested — constraint-level tag parsing in `declare logical schema` DDL is not
  exercised here. Worth a confirming test if constraint-level `with tags` parses.
- **No `apply schema` result rows.** Surfacing `acknowledged: N` directly as
  `apply schema X` output rows remains deferred (the statement is void today;
  converting it is a separate high-blast-radius change). The report + TVF are the
  surface for now.
