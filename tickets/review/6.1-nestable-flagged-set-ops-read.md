## What landed

Generalized the single-level membership-flag invariant to nestable flagged set-ops, **read
path only**. `A union[inA,inSub] (B union[inB,inC] C)` now CREATEs, plans, and reads — it
previously threw at construction (`SET operation column count mismatch: left has 2, right has
4`) because the inner subtree's flags inflated its total arity.

### `planner/nodes/set-operation-node.ts`

- **Module helpers** `dataArity(node)` (recursive: a set-op's data arity is its left operand's,
  bottoming out at the left-most non-set-op leaf) and `flagCount(node)` (`columns.length -
  dataArity`).
- **Constructor arity check** now compares `dataArity(left)` vs `dataArity(right)` — not total
  column counts — so an inner operand's surfaced flags never trip it.
- **`dataColumnCount()` is now public and recursive** (`dataArity(this.left)`). The runtime
  emitter and the write half (6.2) both consume it (6.2 explicitly fixes its `analyzeSetOpView`
  data-col count to call `root.dataColumnCount()`).
- **`hasSurfacedFlags`** getter = own membership OR `leftFlagCount > 0` OR `rightFlagCount > 0`
  (a flag-less outer over a flagged operand still surfaces the inner flags).
- **`buildAttributes` / `getType`** emit `[data] ++ [L flags] ++ [R flags] ++ [own flags]`.
  `leftAttrs`/`leftType.columns` are already `[data ++ L flags]`, so the code spreads them
  verbatim (preserving data attribute ids and inner flag ids), appends `right`'s flags beyond
  the shared data arity, then own flags. `getType().keys` unchanged (`(intersect|except) ?
  leftType.keys : []`) — keys are data-column ColRefs, never a flag at any depth.
- **`computePhysical` own-flag indices shifted** by `leftFlagCount + rightFlagCount` via a new
  `ownFlagBase` getter: `membershipDomains` / `membershipConstantBindings` use `ownFlagBase + i`;
  `membershipFds` uses `totalCols = ownFlagBase + membership.length` so the all-data superkey FD
  yields `key → {every surfaced flag, inner and own}`. The physical surface is still gated on
  `hasMembershipColumns` (own flags) — inner flags' physical surface stays on the inner node
  (not merged onto the outer; that is not required for the read half).

### `runtime/emit/set-operation.ts`

- `dataColCount = plan.dataColumnCount()` (recursive) replaces `plan.left.getType().columns.length`.
- `runWithMembership` reworked to **`runWithSurfacedFlags`**, selected on `plan.hasSurfacedFlags`.
  It buffers each operand's **full** row, keys the probe sets on the DATA columns (storing the
  full row), and emits `[data] ++ [L flags] ++ [R flags] ++ [own flags]`: an operand's flag slice
  when the data tuple is present in it, else `false × flagCount` (default-false); own flags via
  the per-spec `data ∈ branch` probe. Dedup/multiplicity still on data columns only. The plain
  streaming runners are used only when no flag is surfaced (unchanged).

### `planner/building/select-compound.ts`

- **No code change.** `createSetOperationScope` iterates `getType().columns` against
  `getAttributes()` positionally; both now include surfaced inner flags in the same order and
  same length by construction, so an ORDER BY / enclosing view can reference an inner flag. Pinned
  by the `ORDER BY a surfaced inner flag … at depth` test.

### `docs/view-updateability.md`

- New "Nestable flagged set-ops (read half)" paragraph in § Set-operation membership columns
  (projection rule, data-only alignment/dedup/keys, default-false read rule, pointer to the
  write half).

## Honest gaps / where to look hardest (reviewer: treat tests as a floor)

- **Static surfaces on a NESTED view are intentionally NOT covered here.** The read-half tests
  use `select *` + `Object.keys` (not `column_info`/`view_info`) to assert the surfaced layout,
  precisely because the mutation static-surface code (`mutation/set-op.ts` `analyzeSetOpView`,
  `func/builtins/schema.ts` `isSetOpBranchWritable`) still computes `dataColCount = attrs.length
  - flags.length` and would over-count a nested body's surfaced inner flags as data columns. That
  is **owned by the write-half ticket 6.2** (`nestable-flagged-set-ops`), which explicitly fixes
  it. Confirm nothing in the read path calls those surfaces on a nested view (the full suite
  stays green because the binary tests are binary and my nested tests avoid those surfaces) — but
  if a reviewer queries `column_info`/`view_info` on `Vn`, expect inaccurate writability until 6.2
  lands. Verify this is genuinely read-only and not a latent over-claim reachable from a read.
- **NULL data tuples at depth** are not exercised by a dedicated nested test; the single-level
  NULL-probe test passes and the same `dataComparator` (NULL = NULL) is used at every depth, so
  this is covered by construction, not by an explicit case. A nested NULL probe test would be a
  cheap strengthening.
- **Left-operand surfaced flags (`[L flags]` section).** The projection rule and code handle a
  flagged LEFT operand, but via SQL the left operand of a compound is never itself a bare flagged
  `SetOperationNode` (a parenthesized left compound is wrapped as a `select * from (…)` subquery,
  which flattens its flags into data columns). So the `[L flags]` path has **no SQL-reachable
  test** — it is exercised only structurally (leftFlagCount is 0 in every test). If the reviewer
  wants coverage, a hand-built plan node is the only way to drive it; flag whether that is worth
  a unit test or an explicit "not reachable via SQL" note.
- **Default-false soundness** is argued (an output row absent from operand R is in none of R's
  nested branches) and verified for all four outer ops in the agreement/intersect/except/union-all
  tests, but the argument is the load-bearing correctness claim — worth an adversarial re-check
  for `intersect`/`except` as the INNER op (the inner node computes its own flags; the outer just
  surfaces them — tested indirectly via inner `union` only; an inner `except`/`intersect` whose
  flags are surfaced through an outer op is not a dedicated case).
- **No optimizer/dead-flag pruning** for surfaced inner flags (same as the binary case — the
  surfacing runner is selected whenever any flag exists). Correctness unaffected; a `union all`
  with an unused surfaced flag forgoes the streaming runner. Pre-existing limitation, documented.

No `.pre-existing-error.md` was written — the full suite was green before and after.
