## What landed

Removed the four `quereus.update.*` **routing** keys — `target`, `exclude`, `delete_via`, and the `policy` knob — leaving **`default_for.<column>` as the sole retained `quereus.update.*` key**. The override surface is now one mechanism: predicates rule, per-row writable presence/membership columns state routing explicitly (outer-join existence column + set-op membership columns, both delivered by the two prereqs), and `default_for` supplies missing insert values. The prereqs `outer-join-existence-column` and `set-op-membership-write` are both in `complete/`, so their replacement surface is live.

## Reviewer focus / known gaps

1. **Semantic change to scrutinize:** a routing tag on a *single-source* view used to be a documented inert no-op; it is now an `unknown-reserved-tag` **error at mutation time** (pinned in `93.4` `sst_v` and the adversarial-edges section). Confirm this hard-error posture is the intended end state (the ticket's expected-end-state says yes).
2. **Family B conversion is the highest-risk edit.** The inner-join `delete_via=parent` tests became LEFT-join existence-column tests, so the *behavioral contract changed shape* (null-extend vs. hide). Re-derive the fuzz `expView` by hand against `set hasP = false where cc = K` and confirm the "every child appears, parent-deleted children read null-extended" expectation — especially the `routedSeen`/`sharedSeen` guards still trip (they did across 80 runs locally, but the generator bounds are unchanged from the original).
3. **`er_jv` (93.4) relies on `ON DELETE CASCADE` + `foreign_keys` ON** to delete the parent and clear the orphaned child cleanly; the property test instead uses `pragma foreign_keys = false`. Both are valid demonstrations of the same routing, but worth confirming the cascade path is what you'd want documented as the canonical example.
4. **`ctx` is now an unused leading param** in `decomposeDelete` (and `tags` is gone). tsc has no `noUnusedParameters` and eslint uses `args: after-used`, so neither fails — the param is kept only for signature uniformity with `decomposeUpdate`. Drop it if you prefer; harmless either way.
5. **`TypedValueFor`** is now `K extends string ? string : string` (kept as a mapped type so a future enum key can reintroduce a closed union without touching call sites). A reviewer who prefers a bare `string` alias can simplify; no call site depends on the distinction.
6. **Coverage parity, not addition.** Per the ticket the existing routing coverage was *converted*, not extended. The prereqs already ship deep existence-column / membership-write tests (e.g. `property.spec.ts` `rj_ex`), so some Family B conversions overlap that coverage; that overlap is intentional (preserve the named shared-parent invariant), not redundant scope creep.
