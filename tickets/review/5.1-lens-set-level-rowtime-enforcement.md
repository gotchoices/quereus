## What this delivered

A **lock-and-document** ticket: no enforcement logic changed. The row-time
set-level class (a logical `unique` / PK whose key is answered by a **basis
`UNIQUE` + non-stale row-time covering MV**) is enforced end-to-end with **no
dedicated lens code path**, because:

- the prover classifies the key `row-time` *only when* such a basis UC + covering
  MV exists (its own precondition), and
- the single-source spine re-plans the lens write to that **basis table by name**
  (in basis-column terms, renames handled, `onConflict` / `upsertClauses`
  preserved), so the basis INSERT/UPDATE hits the memory layer's
  `checkUniqueConstraints` → `checkUniqueViaMaterializedView` — an O(log n)
  existence lookup that honors `ABORT` / `IGNORE` / `REPLACE`.

So detection **and** conflict resolution fire for free. `collectLensSetLevelConstraints`
correctly returns `[]` for row-time (only `commit-time` emits) and
`rejectLensSetLevelConflictResolution` correctly declines to reject row-time
(it gates only on `hasCommitTimeSetLevelObligation`). Both verified untouched.
