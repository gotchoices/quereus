## What landed

`scripts/check-docs.mjs` grew a fourth check, wired into `main()` between
`checkInvariants` and `checkRatchet`. It validates the shape of the stability labelling,
not its judgement: it cannot know whether lenses really are Experimental, only that
lenses are labelled, that the label names a tier that exists, and that the label agrees
with `docs/.stability.json`.

The success line is now
`Docs OK: links resolve, invariants well-formed, sizes within ratchet, tiers declared.`

### The five rules

1. **Tier vocabulary agreement.** The `###` headings under `## Tiers` in
   `docs/stability.md` and the `tiers` array in `docs/.stability.json` must be the same
   set. Renaming a tier in one place fails, in both directions.
2. **Total classification.** Every `docs/**/*.md` (minus the frozen `review.md` /
   `review.html`) appears exactly once across the `docs` map and the `untiered` array.
   Absent → fail; in both → fail; an entry naming a file that no longer exists → fail
   (unlike the size ratchet, whose orphan entries are deliberately inert). An entry
   naming a frozen review artifact gets its own message rather than the misleading
   "does not exist".
3. **Header banner present and correct.** For a doc in the `docs` map, exactly one
   banner must sit in the *header window* — the lines from the `#` H1 to the first
   subsequent heading of any level, or six non-blank lines, whichever comes first — and
   its tier must equal the recorded tier.
4. **No banner on an untiered doc,** anywhere in the file. `docs/stability.md` is itself
   untiered, and its own example banner is inside a fence, so `stripFences` runs before
   any banner is extracted.
5. **Section banners are checked for tier validity only.** A banner below the header
   window is a section override — that is how `declare schema` is Beta inside a Stable
   `sql.md` — so it is deliberately *not* required to agree with the doc's tier.

A line that starts `> **Stability` but does not match the banner regex exactly is
reported as `malformed stability banner: <the line>` rather than silently counting as
"no banner". The regex is anchored and unforgiving on purpose; the em dash is U+2014.

`docs/.stability.json` is read, never written. There is no `--update-stability`
counterpart to `--update-ratchet`.

### A real defect fixed on the way

`docs/sync.md` begins with a UTF-8 byte-order mark. `readText` did not strip it, so
`/^# /` never matched that file's H1 — meaning its title anchor has been invisible to
Check A (link integrity) all along, and Check D reported it as "no `#` heading". The fix
is a leading-BOM strip in `readText`, which benefits every check. This was not
in the ticket; it is a small in-scope repair to the shared reader, not a doc edit —
`docs/sync.md`'s bytes are untouched.

## How to exercise it

`yarn docs:check` (or `node scripts/check-docs.mjs`) → the success line above, exit 0.

Every negative case below was walked by hand during implementation: make the edit, run
the checker, confirm the message, revert. The tree is clean apart from
`scripts/check-docs.mjs` and `docs/doc-conventions.md`.

| Edit | Message |
| --- | --- |
| delete the banner from `docs/lens.md` | `docs/lens.md: tiered 'Experimental' but carries no stability banner under its H1` |
| retitle `docs/lens.md`'s banner to `Stable` | `docs/lens.md:3: banner says 'Stable' but docs/.stability.json records 'Experimental'` |
| add an unclassified `docs/scratch.md` | `docs/scratch.md: not classified in docs/.stability.json — add it to 'docs' with a tier, or to 'untiered'` |
| add a banner to `docs/architecture.md` (untiered) | `docs/architecture.md:3: untiered doc carries a stability banner` |
| write that banner with a hyphen for the em dash | `docs/architecture.md:4: malformed stability banner: > **Stability: Provisional** - see …` |
| classify `docs/scratch.md` as `Experimental`, give it no H1 | `docs/scratch.md: tiered 'Experimental' but has no '#' heading — cannot locate the header window` |
| give it two banners under its H1 | `docs/scratch.md:4: a second stability banner in the header window — a doc declares exactly one tier` |
| give it a `Provisional` banner under a `## Section` | `docs/scratch.md:10: section banner names tier 'Provisional', which is not defined in docs/stability.md` |
| delete `docs/scratch.md`, leaving its entry | `docs/.stability.json: entry 'docs/scratch.md' names a file that does not exist` |
| put `docs/todo.md` in both `docs` and `untiered` | `docs/.stability.json: 'docs/todo.md' is classified twice — it is in both 'docs' and 'untiered'` |
| add `docs/review.md` to the `docs` map | `docs/.stability.json: entry 'docs/review.md' is a frozen review artifact — every doc check skips it, so it must not be classified` |
| swap `Internal` for `Provisional` in `tiers` | `docs/.stability.json: tier 'Provisional' is not defined in docs/stability.md` **and** `docs/stability.md: tier 'Internal' is defined here but missing from docs/.stability.json` |

A banner shown inside a fenced code block is ignored — verified by putting a
`> **Stability: Nonsense** …` banner in a fence in the scratch doc and seeing no
failure. This is also pinned in `selfTest()`.

`selfTest()` gained: the canonical banner parses to its tier; five near-misses (hyphen
for em dash, no link, lowercase `stability:`, no full stop, a link relative to the wrong
file) all parse to `null`; a fenced banner does not survive `stripFences`; the header
window ends at the first subsequent heading; `headerWindow` returns `null` with no H1;
and `slugify(headingText('Tiers')) === 'tiers'`, so a rename of the `## Tiers` heading is
caught once rather than as forty simultaneous dead-anchor failures.

## Known gaps — treat these as the starting point

- **Nothing was run beyond `yarn docs:check`.** No file under `packages/` is touched, so
  `yarn lint` and `yarn test` are unaffected by construction, and neither was run.
  `yarn check` runs `test:full` and is not agent-runnable. If you want a belt-and-braces
  pass, `yarn lint` is the cheap one — but note `scripts/` is outside every package's
  eslint config, so it lints nothing here.
- **The `selfTest()` cases are the floor, not the ceiling.** They exercise `parseBanner`,
  `stripFences`, and `headerWindow` as pure functions. `definedTiers`,
  `checkClassification`, and `checkTieredDoc` have no pure-function tests at all — they
  were verified only by the hand-walk above, which leaves no artifact. A reviewer who
  wants regression coverage of the rules themselves would have to factor
  `checkStability` to take an injected `{ docs, stability }` rather than reading the
  filesystem.
- **A doc classified in both `docs` and `untiered` takes the untiered branch,** so its
  missing header banner is *not* also reported. The double-classification failure fires,
  which is the actionable one, and behaviour is deterministic — but the second message a
  reader might expect never appears.
- **`readStability()` reads `.stability.json` with a bare `readFileSync`,** matching
  `readBudget()`. A BOM in either JSON file would produce an opaque `JSON.parse` error
  rather than a checker message. Given what we just found in `docs/sync.md`, that is not
  a hypothetical class of problem — but no JSON file in the repo has one today, and
  fixing it is a change to Check C's reader as much as Check D's.
- **`BANNER_ISH` (`/^>\s*\*\*Stability\b/`) will flag ordinary prose** that happens to
  open a blockquote with bold "Stability…". No doc does today. If one ever legitimately
  needs to, the fix is to tighten `BANNER_ISH` to require the colon, not to weaken the
  malformed-banner report.
- **Tripwire, parked as a `NOTE:` at `headerWindow` in `scripts/check-docs.mjs`:** the
  six-non-blank-line bound on the header window has no bite today, because every doc puts
  its banner directly below the H1. If a doc ever needs a longer preamble above its
  banner, raise the bound rather than dropping it — a banner buried in an intro reads as a
  section override, and the doc then fails as though it had no header banner.
