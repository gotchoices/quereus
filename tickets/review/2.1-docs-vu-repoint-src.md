## What was done

four satellites. Code comments across the engine still cited moved sections by the old
file path. This ticket repointed each such comment to the file that now holds the section.

**Comment text only — zero behavioral change.** 37 files, 101 insertions / 101 deletions,
every changed line carries a doc-path token (verified by diffing for any non-doc-path line —
none). `yarn build` green proves no code moved out of a comment.

### Judgment calls the reviewer should sanity-check

- **`§ Set Operations` → `vu-operators.md`, NOT `vu-setops.md`.** `docs/vu-setops.md` exists but
  is a *different* feature ("Set-Operation Membership" columns). The comments citing
  `§ Set Operations` mean per-operator union/intersect/except writability, which lives in
  `vu-operators.md` (its Union All/Union/Intersect/Except headings). There is no single
  "Set Operations" heading now; the loose label was kept as a subject descriptor.
- **`§ CTEs and Subqueries` normalized to `§ Common Table Expressions`** (delete/update/insert/
  dml-target). That old heading was renamed by the split to "Common Table Expressions and the
  CTE-name DML target"; the section text was updated so it doesn't name a dead heading.
- **Two-target comments split.** Where one path cited sections now in different files
  (e.g. `§ Inner Join, § returning` or `§ Inner Join, § Mutation Context`), the ref was split
  so each half points at its home; only the `view-updateability.md` half moved, any
  `docs/lens.md` / `docs/sql.md` / `docs/architecture.md` half was left as-is.
- **`view-complement.ts:9-10`** — the comment reads `§ The predicate-honest complement, the
  § Philosophy fan-out …` mid-prose. Complement moved to `vu-roundtrip.md`; Philosophy stays in
  the overview. Split by inserting an explicit `docs/view-updateability.md` path before
  `§ Philosophy`. Worth a readability eyeball — it's the only edit that threads a path into a
  sentence rather than a `See …` pointer.
