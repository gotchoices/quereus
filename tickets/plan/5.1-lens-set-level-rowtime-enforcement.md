## Context

Sibling of `lens-set-level-commit-time-enforcement`. Where the commit-time class
is a detection-only O(n) scan (no covering structure), the **row-time** class has
a basis covering structure the prover already resolved:

```
{ kind: 'enforced-set-level', mode: 'row-time', structure: CoveringStructureRef }
```

(see `classifyKeyConstraint` â†’ `findBasisCoveringStructure` â†’
`_findRowTimeCoveringStructure` in `lens-prover.ts`; the prover prefers a
non-stale row-time covering MV and records its name on `obligation.structure`).
Classified, not enforced.

## Requirements

- A mutation through the lens that would violate an `enforced-set-level{row-time}`
  logical key must be enforced via an **existence lookup through the basis
  covering structure** named on the obligation â€” O(log n), not the O(n) scan.
- This class **unlocks conflict resolution** through the lens: `insert or replace`,
  `insert or ignore`, and explicit `abort` must resolve against the logical key
  using the covering structure, matching the semantics of a physical UNIQUE
  enforced through a covering MV.
- Compose with row-local checks and the commit-time class on the same write.
- No behavior change for tables without a row-time obligation.

## Design notes (from the implement/review handoff â€” verify during planning)

The machinery to enforce a UNIQUE **through a covering MV** already exists for
physical constraints â€” see `vtab/memory/layer/manager.ts` and the "enforcement
through a covering MV" path documented in `docs/materialized-views.md`
(`covering-structure-mv-rowtime-enforcement` and
`covering-mv-enforcement-prefix-scan-and-preference` shipped it). The work here is
**routing the logical obligation** through that existing path:
  - map the logical key columns â†’ basis columns (reuse the `logicalToBasisColumnMap`
    rewrite in `lens-enforcement.ts`; consider the shared-helper factoring the
    review flagged),
  - select the covering structure the prover named (`findIndexForConstraint`
    prefers the covering MV),
  - thread the statement-level conflict action (`abort`/`ignore`/`replace`) into
    the lookup so the resolution matches physical-UNIQUE semantics.

## Scope boundary
