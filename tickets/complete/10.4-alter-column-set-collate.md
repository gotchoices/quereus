# COMPLETE: ALTER TABLE … ALTER COLUMN … SET COLLATE

`ALTER TABLE t ALTER COLUMN c SET COLLATE <name>` — a sibling of the existing
`SET DATA TYPE` / `SET DEFAULT` / `SET NOT NULL` verbs — changes a column's collation
in place. Collation is projected into the declarative diff catalog so the differ
detects collation drift and emits `SET COLLATE` migrations. Unlike tags, collation is
**semantic**: it changes `=` / `ORDER BY` and participates in the schema hash.

The memory module re-keys / re-sorts every PK / UNIQUE / index that orders by the
column and re-validates uniqueness under the new collation; a set unique under `BINARY`
that collides under `NOCASE` is rejected with `CONSTRAINT` and rolled back. The store
module applies it as a schema-only change (see Review finding #1).

## Data flow

```
SQL ─parse→ alterColumn{setCollation} ─build→ AlterTableNode ─emit(runAlterColumn)→
  validateCollationForType (reject unknown, CREATE-time error shape) →
  module.alterTable({type:'alterColumn', setCollation})
    memory: update ColumnSchema.collation; propagate into PK-def + index columns;
            rebuildAllSecondaryIndexesStrict() then rebuildPrimaryTreeStrict()
            (both throw CONSTRAINT on collision → catch restores prior schema +
             non-strict secondary rebuild)
    store:  schema-only collation update (saveTableDDL)
  → schema.addTable(updated)
diff: catalog.collation (default BINARY) vs declared COLLATE
  → computeColumnAttributeChange sets ColumnAttributeChange.collation on drift
  → generateMigrationDDL emits SET COLLATE (after SET DATA TYPE, before DEFAULT/NOT NULL)
```

   memory rejects the ALTER. Documented today, but a real (narrow) integrity gap. Filed
   `tickets/backlog/store-set-collate-existing-row-revalidation.md` (future work; needs
   store physical re-keying design). When closed, drop `41.7.1` from `MEMORY_ONLY_FILES`.

### Noted, not changed (low-risk, theoretical)
4. **Rollback after primary-tree swap:** the only step after `rebuildPrimaryTreeStrict`
   succeeds is `emitSchemaChange`. If a registered listener threw there, the catch
