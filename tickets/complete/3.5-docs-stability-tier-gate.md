## What landed

`scripts/check-docs.mjs` grew a fourth check, wired into `main()` between `checkInvariants`
and `checkRatchet`. It validates the *shape* of the stability labelling, not its judgement:
it cannot know whether lenses really are Experimental, only that lenses are labelled, that
the label names a tier that exists, and that the label agrees with `docs/.stability.json`.

The success line is now
`Docs OK: links resolve, invariants well-formed, sizes within ratchet, tiers declared.`

### The five rules

1. **Tier vocabulary agreement.** The `###` headings under `## Tiers` in `docs/stability.md`
   and the `tiers` array in `docs/.stability.json` must be the same set. Renaming a tier in
   one place fails, in both directions.
2. **Total classification.** Every `docs/**/*.md` (minus the frozen `review.md` /
   `review.html`) appears exactly once across the `docs` map and the `untiered` array.
   Absent → fail; in both → fail; an entry naming a file that no longer exists → fail
   (unlike the size ratchet, whose orphan entries are deliberately inert). An entry naming a
   frozen review artifact gets its own message rather than the misleading "does not exist".
3. **Header banner present and correct.** For a doc in the `docs` map, exactly one banner
   must sit in the *header window* — the lines from the `#` H1 to the first subsequent
   heading of any level, or six non-blank lines, whichever comes first — and its tier must
   equal the recorded tier.
4. **No banner on an untiered doc,** anywhere in the file. `docs/stability.md` is itself
   untiered, and its own example banner is inside a fence, so `stripFences` runs before any
   banner is extracted.
5. **Section banners are checked for tier validity only.** A banner below the header window
   is a section override — that is how `declare schema` is Beta inside a Stable `sql.md` —
   so it is deliberately *not* required to agree with the doc's tier.

A line that starts `> **Stability` but does not match the banner regex exactly is reported as
`malformed stability banner: <the line>` rather than silently counting as "no banner". The
regex is anchored and unforgiving on purpose; the em dash is U+2014.

`docs/.stability.json` is read, never written. There is no `--update-stability` counterpart
to `--update-ratchet`, because a flag that classifies a doc for you would classify every doc
`untiered`.

### A real defect fixed on the way

`docs/sync.md` begins with a UTF-8 byte-order mark. `readText` did not strip it, so `/^# /`
never matched that file's H1 — meaning its title anchor had been invisible to Check A (link
integrity) all along. The fix is a leading-BOM strip in `readText`, which benefits every
check. `docs/sync.md`'s own bytes are untouched.

## How to exercise it

`yarn docs:check` (or `node scripts/check-docs.mjs`) → the success line above, exit 0.

The implement pass walked every negative case by hand (edit, run, read the message, revert).
The review pass converted the checkable subset of that walk into `selfTest()` assertions, so
the rules now have a regression artifact rather than a table in a ticket.

### Major — none

No finding warranted a new ticket. Check D is a self-contained script with no runtime surface;
its five rules were each exercised, and the two behaviours the handoff called out as
deliberate (orphan entries fail here but are inert in the ratchet; section banners are not
required to match the doc tier) are correct as designed and now pinned by tests.

### Conditional — parked as tripwires, not tickets

- **The banner's link target is the literal `stability.md#tiers`, which only resolves from a
  doc sitting directly in `docs/`.** `docs/images/` holds no markdown, so no doc is affected.
  The first `docs/<subdir>/x.md` that wants a tier will need `../stability.md#tiers` — which
  the banner regex rejects, and which Check A would otherwise happily resolve. Parked as a
  `NOTE:` on `BANNER` in `scripts/check-docs.mjs`, saying to accept both forms at that point
  rather than loosening the anchor.
- **The six-non-blank-line bound on the header window has no bite today,** since every doc
  puts its banner directly below the H1. The implementer's `NOTE:` at `headerWindow` already
  records this and says to raise the bound rather than drop it; re-read and left in place.

### Accepted limits, deliberately unchanged

- **`BANNER_ISH` (`/^>\s*\*\*Stability\b/`) would flag ordinary prose opening a blockquote
  with a bold "Stability…".** No doc does. Tightening it to require the colon would let a
  missing colon read as "no banner at all", which is the failure mode the malformed-banner
  report exists to prevent. The trade is the right way round.
- **A doc classified in both `docs` and `untiered` takes the untiered branch,** so its
  missing header banner is not *also* reported. The double-classification failure — the
  actionable one — fires regardless, and the behaviour is deterministic. Branching to report
  both would add a code path for a state the checker already rejects.
