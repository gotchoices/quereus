## What shipped

The compound grammar accepts a parenthesized query expression `( <query-expr> )`
as a `query-term` operand on **both** legs, and equally as a view body, CTE body,
and top-level query. All three compound sites were unified onto one operand reader
(`parseCompoundOperand`) + one left-associative tail parser (`parseCompoundTail`),
with the operator/membership decode and the trailing ORDER BY/LIMIT consumption
each extracted into a single shared helper. No new AST node: pure grouping unwraps
to the inner `QueryExpr`; a grouped left operand is carried as the existing
`select * from (<inner>) as values_<offset>` subquery wrapper.

Parentheses are the escape hatch into **left-associative** grouping; unparenthesized
chains keep their historical **right-leaning** shape byte-for-byte (a bare operand
greedily consumes the rest of the chain; a parenthesized operand iterates the loop,
wrapping the accumulator when it cannot host the new compound).

### Found + FIXED inline (minor)

**Round-trip non-idempotence for compound `ORDER BY` / `LIMIT` (gap #1 + a related
pre-existing bug).** `selectToString` emitted a compound select's trailing
`order by` / `limit` / `offset` in their *normal* position (before the `union â€¦` leg),
producing `select 1 order by 1 union select 2` â€” which is **un-reparseable** (parse error),
not merely non-idempotent. This was a **pre-existing** defect for *any* compound carrying an
outer ORDER BY/LIMIT (e.g. `select â€¦ union select â€¦ order by â€¦ limit â€¦`), and it silently
broke `generateViewDDL` persistence for such views (the canonical DDL it emits could not be
re-imported). Gap #1 (`A union (select 2 order by 1 limit 1)`, where the leg carries its
own order/limit) was a second facet of the same root cause.

Fix in `ast-stringify.ts` (`selectToString` + new `compoundLegToString`):
- Outer `order by` / `limit` / `offset` on a compound are now emitted **after** the full
  compound chain (their true semantic scope), not in the left-leg position.
- A SELECT leg that carries its **own** `order by` / `limit` / `offset` is wrapped in
  grouping parens so those clauses re-bind inside the leg on re-parse.
- The change is inert for compounds without trailing modifiers (the common case), so all
  pre-existing compound round-trips are byte-identical.

Two regression tests added in `emit-roundtrip.spec.ts â†’ parenthesized compound legs`
(both would have *thrown* before the fix): the leg-local order/limit case, and the
outer-order/limit-after-compound case.
