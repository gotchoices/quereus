## What landed

A shared **backward (maintenance-direction) cost surface** (`planner/cost/index.ts`) plus a
create-time gate (`database-materialized-views.ts`) that makes MV row-time eligibility a cost
choice among structurally-sound strategies via `selectMaintenanceStrategy`, replacing the
implicit shape-allowlist judgement. For the covering-index shape the sound set is
`['inverse-projection']` only, so the gate still always returns the inverse-projection arm —
now annotated with `chosenStrategy` + `sourceStats`. A `MaintenancePlanCommon` base DRYs the
shared identity/cost fields across the three plan arms. Docs scoped `deltaPerRowFallbackRatio`
to the detection kernel (assertions/watchers); MV maintenance reuses it only as the stats-absent
residual fallback. See the implement commit `e55723b2` for the full diff.

### Correctness — checked, sound
- The central invariant ("inverse-projection is never demoted to full-rebuild for an eligible
  shape") is correctly encoded by keeping `'full-rebuild'` **out** of the covering-shape sound
  set. I verified the failure mode it guards against is real: for an empty/small source,
  `forwardBodyCost` (≈ `seqScanCost(0)=0.1`) undercuts inverse-projection's per-row cost (0.4),
  so had `'full-rebuild'` been a competitor, argmin would have picked the unwired arm and the
  defensive guard would have thrown — breaking all MV creation. The gate is robust regardless of
  whether the StatsProvider returns a real `0` (CatalogStatsProvider) or the naive `1000` default.
- `buildMaintenancePlan` is the **only** constructor of a `MaintenancePlan`; the
  `MaintenancePlanCommon` refactor leaves no partial object literals elsewhere. `estimateMaintenanceStats`
  uses the StatsProvider (`tableRows`/`distinctValues`) and `optimizer.tuning` APIs correctly
  (right signatures, `?? DEFAULT_SOURCE_ROWS` for the `undefined` case).
- Layering: `MaintenanceStrategy` in `cost/` is a hand-maintained mirror of `MaintenancePlan['kind']`
  rather than a shared type — **deliberate and correct**, since `planner/cost/` must not depend on
  `core/`. Both definitions currently match. Not a finding.

### Minor — fixed in this pass
- **Doc-accuracy bug in `buildMaintenancePlan`'s JSDoc** (`database-materialized-views.ts:446`).
  The method-level JSDoc contradicted both the actual code and its own inline cost-gate comment on
  the *exact* subtle invariant that already cost the implementer a debugging cycle: it claimed the
  covering-shape sound set was `'inverse-projection'` **and** the `'full-rebuild'` floor (code uses
  `['inverse-projection']` only), and that a body "is rejected at create" via `isFullRebuildPathological`

  sound set is inverse-projection-only (with the small-source reasoning), and that the reject/demotion
  machinery is implemented-but-dormant until the general-bodies arms are wired. Comment-only change;
  typecheck/lint unaffected.
