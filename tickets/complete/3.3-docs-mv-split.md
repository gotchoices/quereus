## What landed

`docs/materialized-views.md` shrank from 28,570 words to ~10,400 — under the 12,000-word cap, so
its ratchet entry in `docs/.doc-budget.json` was removed rather than lowered. Five new documents
carry the extracted subjects, each comfortably under the cap:

| File | Holds |
| --- | --- |
| `docs/materialized-views.md` | Intro, `Why one model`, `Substrate`, `DDL statements`, `Query resolution`, `Write boundary`, `Change-scope projection`, `Declarative-schema integration`, `Current limitations`, a `## Topic documents` table, and seven stubs at the moved sections' original headings |
| `docs/mv-maintenance.md` | `Maintenance strategy` + `Maintenance (row-time, per-statement)` |
| `docs/mv-constraints.md` | `Derived-row constraint validation` + `Covering structures` |
| `docs/mv-backing-host.md` | `Backing-host capability`, `The store host`, `Cross-module atomicity` |
| `docs/mv-ingestion.md` | `External row-change ingestion` and its subsections |
| `docs/mv-schema-change.md` | `Schema-change staleness`, rename propagation, `ALTER TABLE` on the maintained table |

Narrative history was deleted (not archived), three `## Rejected alternatives` sections were
condensed out of the "we used to do Y" passages, and the design prose behind seven of the nine
`## Current limitations` bullets moved to `docs/todo.md § Materialized views`. `## Why one model`
was kept whole.

`yarn docs:check`, `yarn lint`, and `yarn test` are green. `yarn test:full` was not run: the only
source changes are comment text, which the store-backed suite cannot observe.

## Review findings

### Checked

- **Heading survival.** Every `##`/`###`/`####` heading from the pre-split file appears in exactly
  one of the six documents. Verified mechanically against `git show HEAD~1:docs/materialized-views.md`.
  The only additions are the five new titles, the seven stubs, `## Topic documents`, and three
  `## Rejected alternatives`.
- **Prose survival.** Of 310 source lines over 150 characters, 54 do not appear verbatim in the
  six new files. Each was inspected: 40-odd are pure link retargets (`](#anchor)` →
  `](mv-foo.md#anchor)`), the rest are the deliberately-deleted history passages, the limitations
  bullets that moved to `todo.md`, and the two edits made during this review. One was a genuine
  loss — see below.
- **Anchor resolution, both directions.** Every inbound link to a `materialized-views.md#…` anchor
  from anywhere in `docs/` or `packages/` still resolves, and none of them lands on a stub — the
  eleven live inbound anchors all point at real content the overview kept. Every link *into* the
  five satellites resolves too. Confirmed by `yarn docs:check` plus an independent enumeration.
- **Code-comment section references.** Every `docs/mv-*.md § <Section>` reference in
  `packages/*/src` names a heading that exists in the file it names. The one guessed retarget —
  `vtab/backing-host.ts:168`, whose old comment cited a `§ Row-time refresh` that never existed —
  was checked against the code it annotates (the `MaintenanceOp` union) and the new target,
  `mv-maintenance.md § Maintenance (row-time, per-statement)`, is right: that section is where the
  arms consuming those op kinds are described.
- **The one claim whose meaning changed.** The implementer rewrote the replicable-determinism
  limitation bullet to say the gate exists but no host enables it. Verified: the gate fires in
  `core/database-materialized-views-plan-builders.ts:165` on `host?.requiresReplicableDerivations`,
  and the only code in the tree setting that flag `true` is `test/materialized-view-replicable.spec.ts`.
  The rewrite is correct.
- **Tests and lint.** `yarn lint` clean. `yarn test` green apart from one flaky failure described
  under *Pre-existing failure* below.

### Found and fixed in this pass

- **A cross-reference the split broke.** `mv-maintenance.md` ended with "…and the batch ingestion
  seam **below**", but the ingestion seam moved to `mv-ingestion.md`, so there was nothing below.
  Retargeted to a link. `docs:check` could not catch this: it validates links, and this was prose.
- **Content deleted that should have moved to `todo.md`.** The `Coarsened backing keys` section
  used to end with a future-work idea — that the coarsened-key stamp is a natural candidate for a
  lens deploy-report advisory surface, reading the live record rather than persisting. Doc
  conventions send future work to `todo.md`; this was dropped instead. Restored as
  **`todo.md` § Materialized views → Coarsened-key advisory surface**.
- **A limitations bullet that inverted its own meaning.** The rewritten bullet read "A body with
  no provable unique key is rejected at create — no coarsened lineage key, no row identity, no
  materialization", where the em-dash list reads as *consequences*. But the coarsened lineage key
  is a real derivation the overview documents at length; its absence is a *condition* of the
  rejection, not a result of it. Restored the original's conditional phrasing.
- **`docs/architecture.md:118` contradicted the doc it links to.** It described a materialized
  view as "Dual-registered (the backing `TableSchema` plus a `MaterializedViewSchema`)". The
  substrate section says the opposite — one `TableSchema` carrying a `derivation`. The code agrees
  with `materialized-views.md`: `schema/view.ts:64` reads "the former `MaterializedViewSchema`
  dual-registration record is gone". `architecture.md` was the stale one; rewritten to match.
  The implementer flagged this and left it; it is a one-phrase fix in a doc this change should
  have touched, so it was fixed here rather than filed.
- **Nine mislabeled inbound links.** Links whose target had moved kept their old label, so they
  read `[Materialized Views § Covering structures](mv-constraints.md#covering-structures)` — the
  label names a document the link does not open. Repaired in `docs/lens.md` (4),
  `docs/incremental-maintenance.md` (2), `docs/optimizer-fd.md` (1), `docs/schema.md` (1), and
  `docs/lens.md`'s maintenance link (1). Word counts were held flat so the `lens.md` and
  `schema.md` ratchet entries still pass.
- **`docs/doc-conventions.md` cited deleted prose as if it were live.** Its worked example of
  *narrative history* quotes the "Historically 'same module ⇒ one atomic commit'…" passage and
  attributes it to `materialized-views.md` — where it no longer exists, this ticket having deleted
  it. The attribution now says so. The example itself is worth keeping.

### Found and recorded as a tripwire (not a ticket)

- **A green `docs:check` cannot prove the retargeting was complete.** Every moved `##` section left
  a stub behind under its original heading, so `materialized-views.md#maintenance-strategy` and its
  six siblings still resolve. That is the ticket's stub rule working as designed, but it means the
  link checker cannot distinguish a link deliberately left pointing at a stub from one that should
  have been retargeted and wasn't. Verified by hand that no inbound link currently lands on a stub.
  Parked as an HTML `NOTE:` comment above the `## Topic documents` table, where the next person to
  add a link will meet it.

### Considered and deliberately not filed

- **Neither `mv-maintenance.md` (5,894 words) nor `mv-constraints.md` (5,020) is a single subject.**
  Each carries two `##` sections under one title, because promoting both to `#` would mint two h1s.
  Both are far under the 12,000-word cap and the seam is already visible in the heading structure,
  so a further split would be churn today. `debt-docs-shrink-remaining-megadocs` in `backlog/`
  already owns the general "docs are too long" work if either grows.
- **No content was audited against the implementation.** The ticket scoped this as a move, and the
  prose is preserved verbatim except where noted. Six unaudited documents is what six unaudited
  documents were before. `3.5-docs-invariants-mv.md` in `implement/` is the ticket that extracts
  normative invariants and checks them against the code; that is where the audit belongs, and it is
  already queued. Not re-filed.
- **`docs/architecture.md` gained no pointer to the five satellites.** It links to the overview,
  which now carries the `## Topic documents` table, so the pointer is one hop away and correct.
  `packages/quereus/README.md` lists all five directly. No change needed.
- **The `docs-stability-tiers` ticket will want a banner slot in five new files.** Each satellite
  opens `# Title` → abstract → content, so a banner has an obvious home. That ticket owns it.
